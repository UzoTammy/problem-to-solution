Object Oriented Program

Introduction to OOP

Object-Oriented Programming (OOP) is a way of writing computer programs using the idea of "objects" to represent data and methods. It's like creating a mini-program (an object) for each piece of your program, where each object can contain information (data, like numbers and text) and actions (methods, like calculating or modifying that data).

Imagine a video game with different characters. Each character can be an object. The data for each character might include their health, strength, and speed, while the actions might be things like walk, jump, or attack. Each character (object) can have its own data and actions, but they all belong to the broader game (program).

In OOP, we use "classes" to define the blueprint of these objects, like a template for what data and actions each object will have. Then we create "instances" of these classes, which are the actual objects we use in our program. OOP is popular because it helps organize and manage complex programs by breaking them down into smaller, manageable, and reusable parts (objects). It also makes it easier to keep track of all the data and actions related to each part of your program.

Program 1.2 - Naming Classes

In python, using a clear naming convention is essential for readability and maintenance. While not mandatory, it's highly recommended to adopt camelcase style for class names. This involves capitalizing each word in the name to enhance clarity and distinction. For example, a class handling bank accounts should be named 'BankAccount', a simple coordinate class would be 'Point', and a group focused on young men's fellowship would be aptly named 'YoungMenFellowship'. This naming strategy not only improves code readability but also precisely reflects the purpose and functionality of each class.

Program 1.3 - Instance Variable

Classes in python become truly dynamic and potent when initializing their attributes. This is achieved using the special or 'dunder' method `__init__()`. This method activates each time the class is instantiated. Within `__init__()`, attributes are set based on the arguments passed. Notably, the first argument is always reserved for the class instance itself, commonly referred to as 'self'. While using 'self' is a popular convention, it's not obligatory. This mechanism of attribute initialization is key to harnessing the full power and versatility of classes in object-oriented programming.

  • class Point:
  • def __init__(self, x, y):
  • self.x = x
  • self.y = y

This class has two attributes, self.x and self.y and are respectively assigned x and y arguments. Remember the first argument is reserved for the instance of the class

The first argument as earlier mention is reserved for the class but must not bear the name self. You can name it whatever you choose to however, self is a standard. At this stage do not forget it as you initialize the class (in other languages like JS, Java the initialize is called class constructor)

Let us use what we have initialized, point is declared as an instance of the class Point (please mind the upper and lower cases). Since, the class is initialized, the class must be called with the arguments which automatically becomes the attributes. So we can conclude that the init method transforms arguments into attributes

  • point = Point(2, 3)
  • print(point.x) --> 2
  • print(point.y) --> 3
  • print(point.x + point.y) --> 5

Program 1.5 - Class Variable

Class variables are a fundamental aspect of OOP in Python, allowing for the sharing of values and states across all instances of a class. Understanding how to properly define, access, and modify these variables is crucial for effective object-oriented programming.

  • class Person:
  • num = 10 #defined within the class but outside
  • # of the __init__ or any other method
  • person = Person()
  • person.num -> 10 # can be accessed through the instance of the class
  • Person.num -> 10 # and also through the class itself
  • # it is tempting to change the variable through the instance
  • person.num = 20 # this only changes for this instance
  • person.__dict__ -> 20
  • Person.__dict__ -> 10
  • Person.num = 20 # changes the class variable
  • Person.num -> 20 # proves it

  • # Lets take a practical application of the class variable

  • class Person:
  • N = 0 # number of instances created
  • def __init__(self, name):
  • self.name = name
  • Person.N += 1 # each time an instance in created, add 1 to N
  • ########
  • person1 = Person('Frank')
  • Person.N - > 1
  • person2 = Person('Susan')
  • Person.N -> 2

Class Inheritance

Definition

Inheritance is a fundamental concept in Object-Oriented Programming (OOP), in Python. It allows for the creation of a new class that inherits attributes and methods from an existing class.

Inheritance in OOP is a mechanism where a new class, known as a subclass or derived class or child class, inherits properties (attributes) and behaviors (methods) from an existing class, known as a superclass or base class or parent class.

The primary purpose of inheritance is to promote code reuse and establish a hierarchical relationship between classes.
